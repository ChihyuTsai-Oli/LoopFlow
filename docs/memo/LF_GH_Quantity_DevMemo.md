# LF_GH_Quantity — GH 數量計算模組開發備忘錄

> 發想日期：2026-06-11
> 最後更新：2026-06-11

---

## 一、背景與動機

室內設計工程數量計算目前依賴人工量測與手動填表，流程耗時且容易出錯。本模組目標是透過 Grasshopper 直接讀取 Rhino 場景中的物件幾何與屬性，自動計算數量並填入人工建立的 Excel 表單，輸出可交付的工程數量計算書。

---

## 二、模組定位

```
主要目的：輸出數量計算表單（Excel）
查核流程：輸出前的確認手段，不是終點
```

GH 模組不負責產生表單結構，只負責根據人工建立的表單項目，去場景比對對應物件、計算數量、填回表單。

**與 Registry.json 的關係：完全無關**
本模組直接讀取 Rhino 場景，不經過 Registry.json，兩者資料來源不同，職責完全分開。

---

## 三、建模前提

- 所有物件為**有厚度的獨立實體模型**（含油漆粉光層）
- 模型已建立正確開口，門窗以 Block 置入
- 裝修材（磁磚、石材、油漆等）與結構體各自獨立建模
- 幾何計算直接取表面積／長度，不需要額外的布林運算

---

## 四、資料來源

```
Rhino 場景
├── 物件幾何         → 計算面積、長度、數量、體積
└── UserText 屬性
    ├── _01_空間名稱  → 決定列在哪個空間
    ├── _02_建構狀態  → 區分結構（Existing）與裝修（New）
    ├── _03_ID編號   → 比對表單項目
    ├── _04_ID名稱   → 項目名稱
    └── _08_單位     → 驅動計算方式
```

`_01_空間名稱` 由 LF_Nexus 透過空間範圍 curve 寫入物件 UserText。

## 六、計算規則

`_08_單位` 為唯一驅動來源，字典已完整定義，不需要額外的 `_type_` 欄位：

| 單位 | GH 計算方式 |
|------|-----------|
| 坪 | 表面積 ÷ 3.305785 |
| 才 | 表面積 × 10.7639 |
| cm / mm | 最長邊長度 |
| 組 / 台 / 片 / 樘 | 數量計算 |
| m3 | 體積 |

## 七、表單結構

以工程類別為主軸，空間為次分類：

```
-- 木作工程 --
客廳  天花板    CL-02  輕鋼架天花  __ 坪
客廳  電視牆    WL-11  裝飾壁板    __ 坪
臥室  天花板    CL-02  輕鋼架天花  __ 坪
臥室  衣櫃      CB-07  木紋櫃體    __ mm

-- 水電工程 --
廚房  220V插座  EL-05  插座面板    __ 組
客廳  110V插座  EL-05  插座面板    __ 組
```

表單項目由**人工建立**，變數太多難以無缺漏地預先自動產生。

---

## 八、資料流

```
人工建立 Excel 表單
（工程類別、空間、_03_ID、項目名稱）
        ↓
GH 讀取 Excel 表單
        ↓
依條件比對場景物件
（_03_ID + _01_空間名稱）
        ↓
無法自動比對者 → 人工在 GH 中手動指定
        ↓
_08_單位 驅動計算方式，執行幾何計算
        ↓
填入數量 → 輸出完整 Excel
```

---

## 九、例外處理

自動比對無法完整覆蓋所有項目，手動例外機制作為補充：

- 在 GH 介面中選取物件後手動指定對應的表單項目
- 例外設定**不寫回 Rhino**，只在本次計算階段有效
- 例外處理針對**個別物件**，不影響字典圖層通則

---

## 十、檢核機制

**目的：從場景端反向確認表單的完整性**

```
場景中所有 _02_建構狀態 = New 的物件
        ↓
已被 GH 選取（列入計算）的物件
        ↓
兩者相減 = 未被選取的物件
        ↓
Highlight 顯示於 Rhino Viewport
→ 提示這些物件尚未列入數量計算
```

檢核範圍限定 `_02_建構狀態 = New`，排除結構體（Existing）避免誤報。

## 十二、待處理 Bug

### 跨空間物件的空間名稱歸屬問題

**現況：** LF_Nexus 依 curve 範圍寫入 `_01_空間名稱`，同一物件只能對應一個空間。

**問題：** 橫跨兩個空間的物件（隔間牆兩側、延伸天花板等）會歸屬錯誤或遺漏，導致數量表單的空間分類不正確。

**影響範圍：** 數量表單的空間分類正確性、自動比對的準確率。

**待討論解法：**
- [ ] 切割物件（建模時沿空間邊界切開）
- [ ] 多值欄位（`_01_空間名稱` 支援多個空間）
- [ ] 手動指定例外空間歸屬
