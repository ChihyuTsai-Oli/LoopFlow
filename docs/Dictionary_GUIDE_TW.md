## 1. 字典是什麼

`LoopFlow_Dictionary.xlsx` 是 LoopFlow 系統的**預設屬性定義表**。

每一列代表一個 Rhino 子圖層，預先定義該圖層上的物件應套用哪些 UserText 屬性值（ID 編號、名稱、單位、高程基準等）。執行 `LF_Nexus` 時，系統會讀取這份字典，一鍵將預設值寫入選定物件，確保全案屬性一致。

---

## 2. 檔案格式與放置位置

| 項目 | 說明 |
|---|---|
| 檔案名稱 | `LoopFlow_Dictionary.xlsx`（可在 `_LoopFlow_Config.py` 修改） |
| 放置位置 | **與 `.3dm` 專案檔放在同一個資料夾**，LF_Nexus 會自動尋找 |
| 格式要求 | 僅支援 `.xlsx`；不接受 `.csv` 或 `.xls` |
| 工作表 | 使用第一張工作表（sheet name 不限） |
| 第 1 列 | 版本標題列，例如 `LoopFlow Dictionary v1.0`（程式會跳過此列） |
| 第 2 列 | **欄位標題列**，欄名必須完全符合下方定義，順序可調整 |
| 第 3 列起 | 每列代表一個 Rhino 子圖層的預設屬性 |

## 3. 欄位說明

### 主鍵欄（必填）

| 欄名 | 說明 |
|---|---|
| `__Rhino Layer` | **唯一索引鍵**。Rhino 子圖層的完整路徑，格式見[第 4 節](#4-圖層命名規則)。空白或重複的列會被跳過。 |

### 通用屬性欄

| 欄名 | UserText 鍵 | 說明 | 可用值 / 格式 |
|---|---|---|---|
| `_01_空間名稱` | `_01_空間名稱` | 物件所在空間名稱。通常留空，由 LF_Nexus TagTrigger 依空間邊界自動填入。 | 文字，例如 `主臥室` |
| `_02_建構狀態` | `_02_建構狀態` | 施工狀態。結構類預設 `Existing`，其餘預設 `New`。 | `New` / `Existing` / `Demolished` / `Relocated` |
| `_03_ID編號` | `_03_ID編號` | 材料/工項的唯一代碼，用於工程估算與圖說索引。 | 格式 `XX-NN`，例如 `EX-01`、`CL-01` |
| `_04_ID名稱` | `_04_ID名稱` | 人可讀的材料名稱，用於圖說說明欄。 | 文字，例如 `磁磚地坪` |
| `_05_寬度W` | `_05_寬度W` | 寬度預設值（選填）。 | 數值 |
| `_06_深度D` | `_06_深度D` | 深度預設值（選填）。 | 數值 |
| `_07_高度H` | `_07_高度H` | 高度預設值（選填）。 | 數值 |
| `_08_單位` | `_08_單位` | 數量計算單位，影響工程估算報表。 | `坪` / `才` / `cm` / `mm` / `m3` / `樘` / `組` |
| `_09_實作數量` | `_09_實作數量` | 實際施作數量，通常由 LF_Nexus 計算後填入。 | 數值 |
| `_10_高程基準` | `_10_高程基準` | 高程量測的基準面縮寫，影響 LF_Nexus 高程自動計算邏輯。 | 見下方說明 |
| `_11_高程計算` | `_11_高程計算` | 計算後的高程值（程式自動填入，字典中通常留空）。 | 數值 (cm) |
| `_12_UUID` | `_12_UUID` | 物件唯一識別碼（程式自動產生，字典中通常留空）。 | 自動生成 |
| `_13_備註` | `_13_備註` | 人工備註欄，可說明特殊施工條件或材料規格。 | 自由文字 |

#### `_10_高程基準` 可用值

| 縮寫 | 全名 | 說明 |
|---|---|---|
| `BH` | Bottom Height | 從物件**底面**量取高程 |
| `TH` | Top Height | 從物件**頂面**量取高程 |
| `CH` | Ceiling Height | 從**天花板底面**量取高程（燈具、天花材料適用） |
| `TH/BH` | Top + Bottom | 同時記錄頂面與底面高程（例如樓板） |

### 系統櫃板材欄（`_CB.*`）

> ⚠️ **TW 版使用中文 key 名**，必須搭配 TW 版 `LF_Cabinet_Suite.py` 使用。若與 EN 版腳本混用，CB 欄位將無法正確同步。

| 欄名 | UserText 鍵 | 說明 |
|---|---|---|
| `_CB.01_板材類型` | `_CB.01_板材類型` | 板材分類，例如 `側板`、`門片`、`頂板` |
| `_CB.02_長度L` | `_CB.02_長度L` | 板材最長邊 (cm) |
| `_CB.03_寬度W` | `_CB.03_寬度W` | 板材第二長邊 (cm) |
| `_CB.04_厚度T` | `_CB.04_厚度T` | 板材厚度 (cm) |

## 4. 圖層命名規則

`__Rhino Layer` 欄的值對應 Rhino 圖層樹狀結構，格式為：

```
NN_Category_中文名稱::EnglishType.中文名稱
```

- `NN`：兩位數字排序前綴（`00`、`01`、`02`...）
- `Category`：英文類別名稱
- `中文名稱`：繁中顯示名稱
- `EnglishType.中文名稱`：子圖層，英文類型加中文說明

**完整範例：**

```
02_Wall_牆面::Tiles.磁磚
│                │
│                └─ 子圖層：英文類型.中文說明
└─ 父圖層：編號_英文_中文
```

> **特例：** `20_DW`（門窗）為單層結構，不使用 `::` 子圖層。門窗物件以 Block 形式放在此圖層，資料寫在 Block，不寫在子圖層。

## 5. 撰寫規則與注意事項

### 新增一列

1. 在 Excel 中新增一列
2. `__Rhino Layer` 填入 Rhino 中**完整的子圖層路徑**（大小寫一致）
3. `_02_建構狀態`：結構類填 `Existing`，其餘填 `New`
4. `_03_ID編號`：依類別前綴命名（CL-XX / WL-XX / FL-XX...）
5. `_08_單位`：面積用「坪」，線性用「cm」，體積用「m3」
6. 尺寸欄（`_05_` / `_06_` / `_07_`）若無固定值可留空
7. `_CB.*` 欄：系統櫃板材由腳本自動計算，字典中保持空白

### 禁止操作

| 禁止操作 | 原因 |
|---|---|
| 刪除或改名第 1 列（版本標題列） | `DICTIONARY_SKIPROWS = 1` 固定跳過第 1 列 |
| 改名欄標題 | LF_Nexus 以欄名前綴辨識欄位，改名導致欄位被忽略 |
| `__Rhino Layer` 欄填入重複路徑 | 後出現的列會覆蓋前面的列 |
| 使用 `.csv` 或 `.ods` 格式 | 僅支援 `.xlsx` |

## 6. 字典在 Rhino 中的使用方式

以下流程均透過 **`LF_Nexus`** 主介面執行：

| 操作 | 功能 |
|---|---|
| **Dict. to Layer** | 依字典 `__Rhino Layer` 欄在 Rhino 建立所有父/子圖層 |
| **Dict. to Object** | 選取物件後，依其所在圖層查找字典，寫入對應的 UserText 屬性 |
| **Rhino to XLSX** | 從 Rhino 物件的 UserText 反向匯出，更新 `.xlsx` 字典 |
| **Dictionary Editor** | 自動開啟同目錄的 `LoopFlow_Dictionary.xlsx` 供直接編輯 |

## 7. 受字典影響的檔案

### 核心依賴

| 檔案 | 角色 |
|---|---|
| `_LoopFlow_Config.py` | 定義字典檔名、主鍵欄名、跳過列數 |
| `LF_Nexus.py` | 讀取字典全部欄位；執行三向同步；高程計算依賴 `_10_高程基準` |
| `LF_Dictionary_Editor.py` | 自動定位並用 Excel 開啟字典 |

### 間接依賴（讀取字典寫入的 UserText）

| 檔案 | 依賴欄位 |
|---|---|
| `LF_TAG-O.py` | `_01_空間名稱` |
| `LF_Cabinet_Suite_TW.py` | `_CB.01_板材類型`、`_CB.02_長度L`、`_CB.03_寬度W`、`_CB.04_厚度T` |

## 9. 中途切換字典版本（不建議）

> ⚠️ **不建議在專案進行中切換字典語言版本。** 一旦開始使用某一版本字典，應在整個專案生命週期中保持一致。

TW 版與 EN 版字典的欄位 key 名稱不同（例如 `_01_空間名稱` vs `_01_Space Name`）。若中途切換，舊物件上的 UserText key **不會自動刪除**，導致物件同時帶有新舊兩組 key。`LF_TAG-O` 等以前綴掃描全部 key 的工具，會因此發生不穩定的讀取行為。

**如果非要切換，請依照下列步驟進行：**

1. 執行 LF_Nexus **Dict. to Layer**，建立新版字典的圖層結構
2. 將所有物件搬移到對應的新版圖層
3. **批次清除所有物件上的舊版 UserText key**（需另寫 Rhino Python script，目前無內建工具）
4. 執行 LF_Nexus **Dict. to Object**，將新版字典預設值寫入物件
5. 重新執行 **TagTrigger**，讓 `_01_` 欄位在新 key 名稱下重新填入

> **注意**：步驟 3 是目前 LoopFlow 工具箱尚未提供的功能，需自行撰寫腳本或手動逐物件處理。
